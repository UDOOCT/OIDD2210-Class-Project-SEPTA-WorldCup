═══════════════════════════════════════════════════════

  L1. main.py SLSQP runtime: global 1586-var SLSQP takes ~14s/iteration.
      Measured: 10 iterations = 138s; 500 iterations = ~2 hours.
      Initial point is now budget-feasible ($315K ≤ $350K) ✓ but problem
      structure (2354 constraints) makes each iteration inherently expensive.
      Workaround: use _run_optimization.py for v1 results (greedy + per-line
      warm-started SLSQP, runs in ~15 seconds).
      Root cause: global SLSQP (1586 vars) vs. decomposed per-line approach.

  L2. Equity violations (8 lines): correct behavior under $350K budget.
      Interpretation: SEPTA cannot afford to run extra late-night trains on all
      13 lines simultaneously. The $800K equity penalty dominates the objective.

  L3. BSL clearance = 225 min (3h45m): realistic given 45K post-game fans and
      BSL capacity of 3,060 pax/slot. Not a bug — matches queue math.

  L4. main.py --mode bilevel and --mode sensitivity: also slow (bilevel runs
      upper_solve repeatedly; sensitivity runs 200 trials × SLSQP each).

  L5. Equity in greedy RR allocator is post-hoc, not enforced.
      Documented in formulation.md §16.

  L6. BSL demand model uses Normal distribution for post-game wave.
      Actual stadium egress is more complex (documented in README limitations).
